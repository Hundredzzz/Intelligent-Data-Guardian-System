# 数据安全员 AI 审核测试样本

使用账号：

```text
security / 123456
```

进入“AI审核”页面后，可以：

1. 复制 `01-customer-release-text.txt` 中的拟发布内容到文本框，点击“审核文本”。
2. 上传 `02-finance-file-review.txt`、`03-contract-review.txt`、`04-technical-document-review.txt`，点击“审核文件”。

建议选择对应下拉项：

- 客户发布：外部信息发布审核 / 客户资料 / 对外发布
- 财务资料：内部通信内容检查 / 财务信息 / 部门间共享
- 合同复核：合同资料复核 / 合同内容 / 仅内部流转
- 技术文档：技术文档脱敏检查 / 技术资料 / 部门间共享

## 普通员工审核和数据安全员 AI 审核的区别

普通员工检测审核：

- 员工提交文本或文件。
- 系统先做规则检测，例如手机号、身份证、邮箱、银行卡、敏感词。
- 系统生成检测任务、检测结果、风险预警和主管审核任务。
- 部门主管在“审核申请”中通过或驳回。
- 员工在“检测结果与历史记录”查看审核结果。

数据安全员 AI 审核：

- 数据安全员直接输入文本或上传文件。
- DeepSeek 根据审核场景、资料类型、发布范围进行专业复核。
- 输出结构化建议，包括风险等级、是否建议发布、风险依据、修改建议、审核结论。
- 适合安全员做二次复核、发布前抽检、敏感文档专项检查。
- 不会替员工创建检测任务，也不会走主管审批流程。
